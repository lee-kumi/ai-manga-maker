# ナレッジ フォルダの案内板（_RULE.md）

## このフォルダは何か

**事業の思想・ノウハウ・制作ルールをまとめて置く「考えの土台」。**
「なぜそうするか」を言葉にして残す場所。AIに相談するときも、ここを読ませれば話が早い。

## 中に入れるもの

| 種類 | 例 |
|---|---|
| 思想・方針 | 「漫画は絵より物語」「表紙はアートより広告」など、判断の軸 |
| 制作ルール | セリフは句点を使わない・色数は3色以内・キャンバスサイズ 等 |
| ノウハウ | うまくいった型・失敗したパターン・気づき |

## ルール

1. **「思想（なぜ）」と「実装（どうやる）」を分けて書く**
2. 1テーマ＝1ファイル。あとから探しやすい名前をつける
3. ルールを変えたら、**元にしているツールやスキルにも同時に反映する**（片方だけ直さない）
4. 迷ったら「次にこのフォルダを開くAIや自分が、これで正しく動けるか？」を基準にする

## この事業での実体（原本の場所）

このリポジトリでは、ナレッジの原本はすでに `docs/` にある。ここはその入口。

| 原本ファイル | 中身 |
|---|---|
| `docs/knowledge-base.md` | 全ツールの思想・ノウハウを1か所に集めた総まとめ（地図） |
| `docs/manga-maker-rules.md` | AI漫画メーカー v2 の思想＋制作ルール（漫画本文の原本） |
| `docs/cover-maker-rules.md` | Kindle表紙メーカーの思想＋制作ルール（表紙の原本） |

漫画本文・表紙のルールは上記が正。食い違ったら原本を優先する。
