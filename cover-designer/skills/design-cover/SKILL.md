# SKILL: Kindle表紙設計（STEP0〜5）

## 役割
本の情報を受け取り、「売れる広告」としてのKindle表紙を STEP0〜5 の順で設計し、
画像生成AI用プロンプトと Canva 仕上げ指示までを1枚の設計書にまとめる。

## 動くきっかけ
- 新しい本・漫画の表紙を作りたいとき
- 「表紙を設計して」「売れる表紙にして」「表紙タイプを判定して」と頼まれたとき

## 手順（動詞で3〜7歩）
1. 読む：原本 `../../docs/kindle-cover-design-rules.md` と knowledge/design-source.md を読む。
2. 受け取る：本の情報（タイトル案・ジャンル・テーマ・ターゲット読者）を確認する。足りなければ1つずつ聞く。
3. 分析する：STEP0 売れる要素 → STEP1 感情フック → STEP2 表紙タイプ(A〜D)判定 を出す。
4. 設計する：STEP3 レイアウト（視線誘導・配色3色以内・情報量）を決める。
5. 生成する：STEP4 画像生成プロンプトを「文字なし版」「文字入り想定版」の2つ作る（1600×2560px・日本語文字なし）。
6. 指示する：STEP5 Canva仕上げ（タイトル/サブ/著者名の配置・帯・バッジ・配色・フォント）を書く。
7. 書き出す：設計書を `output/日付_本タイトル_cover.md` に保存する。

## 参照するナレッジ
- `../../docs/kindle-cover-design-rules.md`（原本・最優先）
- `../../Agent.md`（自己完結版の指示書）
- `knowledge/design-source.md`（早見表・約束事）

## 成果物の形と保存先
- STEP0〜5 が縦に並んだ設計書 Markdown 1枚。保存先＝`output/日付_本タイトル_cover.md`

## 合格条件（数えられる形）
1. STEP0〜STEP5 の**6項目すべて**が埋まっている。
2. STEP4 の画像プロンプトが「**文字なし版**」「**文字入り想定版**」の**2つ**ある。
3. 約束を守っている：**色数3色以内** かつ **サイズ1600×2560px明記** かつ **句点なし**。

## 正解例（抜粋・B型 問題解決型の場合）
```
STEP2 表紙タイプ：B型（問題解決型）— 副業ジャンルでベネフィット訴求が刺さるため
STEP3 レイアウト：上=タイトル特大／中央=主役（PCと笑顔の人物）／下=感情フック帯
        配色3色＝ベース#1B3A5B・文字#FFFFFF・アクセント#E8612A
STEP4 画像プロンプト（文字なし版）：a confident person smiling at a laptop, warm light, ...
        no text, 1600x2560px, vertical book cover
STEP5 Canva：タイトル「月3万円の始め方」を上部30%に白特大／著者名を右下小さく／
        アクセント帯に「未経験OK」バッジ（句点なし）
```
