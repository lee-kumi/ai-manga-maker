# Kindle表紙のコンセプト・プロンプト作成（旧3案方式・レガシー）

旧Kindle表紙メーカー準拠の3案方式で、表紙案と画像生成プロンプトを作る。
**現行の推奨は `/cover-design`（STEP0〜5・最適解1案方式・`docs/kindle-cover-design-rules.md` 準拠）**。ユーザーが「3案ほしい」「案を比べたい」と言ったときだけこちらを使う。

## 手順

1. まず `docs/cover-maker-rules.md` を読む。**この文書が唯一のルールの原本**であり、本スキルの記述と食い違う場合は文書を優先する
2. どちらのプロファイルかを確認する：
   - 🎨 **AI漫画の表紙**（イラスト・キャラ主体）
   - 📖 **テキスト本の表紙**（タイトル・タイポグラフィ主体）
3. 本の情報を受け取る（不足していれば聞く）：タイトル・ジャンル・ターゲット読者・一番伝えたいこと（フック）
4. 文書の鉄則に従い、表紙案を3案 →ユーザーが選んだ案の画像生成プロンプトを出力する

## 絶対に守るルール（文書の要点）

- 表紙はアートではなく**広告**。サムネイルサイズで成立するかを最優先
- タイトル・コピーに句点（。）を付けない／色数は3色以内
- 表紙サイズは **1600×2560px**（漫画本文の1000×1414pxとは別）
- 画像に文字は入れさせない。**文字は後からCanvaで載せる**前提でプロンプトを書く
- 漫画表紙はキャラクターシートを参照画像として添付させる（顔ブレ防止・本文との画風統一）

## 各案に含めること

コンセプト／メインビジュアル／タイトルの見せ方／キャッチコピー／配色（3色以内・具体色）／クリックされる理由（ターゲット心理から）
