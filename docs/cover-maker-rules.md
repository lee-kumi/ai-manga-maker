# Kindle表紙メーカー（Cover Compass）基本思想＋制作ルール

この文書は、Kindle表紙メーカー（`cover-maker.html`）に組み込まれている思想とルールを、人間が読める形でまとめたものです。
アプリを改修するとき・「このアプリはルールを満たしているか？」を照合するとき・別のAIに相談するときの基準にします。

- 対象アプリ：https://lee-kumi.github.io/ai-manga-maker/cover-maker.html
- 抽出元：`cover-maker.html` の `COVER_KNOWHOW` 定数とプロンプト生成コード（2026-07-15時点）
- **表紙設計の思想の原本**：`docs/kindle-cover-design-rules.md`（汎用ルール。食い違う場合はそちらを優先）
- 姉妹アプリ：AI漫画メーカー（`v2.html`）のルールは `docs/manga-maker-rules.md` を参照

---

## 基本思想

**表紙はアートではなく「広告」である。**

- 目的は、スクロールを止める・クリックされる・内容が伝わる・購入候補に入ること
- 美しさだけで判断しない。常に**「この表紙は売れるか」**で判断する
- このアプリ単体で表紙を完成させるのではない。**表紙案を出すプロンプト**と**画像生成AIに渡すプロンプト**を設計する場所である
- 文字（タイトル・帯）は画像生成に頼りきらず、**後からCanvaで載せる・整える**（日本語崩れ防止）
- 案は複数並べてユーザーに選ばせない。**プロとして判断した最適解1案に絞る**

## 3つのモード

| モード | 入力 | 向いている人 |
|---|---|---|
| 🪄 おまかせモード | 4項目だけ（タイトル・ジャンル・内容一言・雰囲気）。足りない部分はAIが補完 | まず形にしたい初心者 |
| 📝 詳細モード | 13項目でしっかり指定 | イメージが固まっている人 |
| 🖼 画像から真似る | 好きな表紙画像を添付し、プロンプトを逆算 | 参考にしたい表紙がある人 |

共通機能：作品ごとの自動保存（localStorage・`cm_`系キー）／全作品のバックアップ保存・復元／作品専用GoogleドキュメントURL登録／生成結果の作品別保存。

## COVER_KNOWHOW（全プロンプト共通で入るノウハウ）

`COVER_KNOWHOW` 定数と一致させること。内容は `docs/kindle-cover-design-rules.md` の汎用ルールを反映：

1. **基本思想** — 広告である・売れるかで判断・句点（。）を付けない
2. **視線誘導の順序** — タイトル → 主役 → 感情フック → 補足情報。タイトルが読めない表紙は失敗
3. **感情フック** — 最も感情が動く要素を中心に。情報より感情を優先
4. **読者メリット** — 読者は本ではなく「未来」を買う。何が得られるかを配置
5. **情報量の調整** — 理想は「タイトル＋感情フック＋読者メリット、以上」
6. **表紙タイプ判定** — A人物ドラマ型／B問題解決型／C権威型／D憧れ型を判定してから設計
7. **初心者向けノウハウ** — 色数3色以内（ベース＋文字＋アクセント）／帯・バッジの使いどころ／ジャンル別の画風目安／Canvaで修正しやすい構図

## 出力のルール（通常モード）

アプリは3種類のプロンプトを出す。

1. **GPT表紙案プロンプト**（→ Claude / ChatGPT）
   - まず【STEP0：この本の分析】（固有価値・読者が読む理由・最優先事項・感情フック抽出・表紙タイプ判定）を必ず先に行わせる
   - 次に【STEP1：表紙の最適解】を**1案だけ**出させる（コンセプト／ビジュアル／文字配置・配色・フォント／コピー類（句点なし）／スマホサムネイル評価／避けるべき表現／画像生成プロンプト）
   - 最後に【迷った時の調整案】を短く付けさせる
2. **文字なし画像プロンプト**（→ ナノバナナ / ChatGPT）
   - 文字を一切入れない縦長イラスト。タイトルを載せる余白を残す
3. **文字入り画像プロンプト**（→ ナノバナナ / ChatGPT）
   - タイトル・サブタイトル・著者名まで入った「まずそのまま使える表紙」。Canvaで直しやすい構図を指示

「画像から真似る」モードは、参考表紙の分析＋（本の情報があれば）自分の本用に置き換えたプロンプトを出す。

## サイズのルール

- 表紙キャンバスは **1600×2560px（縦横比約1:1.6）** を基準とする（Kindle推奨）
- 漫画本文（コマ）の 1000×1414px（A4縦）とは別。**表紙だけこのサイズ**を使う

- 逆に、この文書でルールを変えたら、`cover-maker.html` にも反映する
- 思想レベルの変更は `docs/kindle-cover-design-rules.md`（原本）から直し、この文書・アプリ・`Agent.md`・スキルへ展開する
