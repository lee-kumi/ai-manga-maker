# Kindle表紙設計AI 汎用ルール（原本）

この文書は、Kindle表紙を「売れる広告」として設計するための汎用ルールの**原本**です。
スキル（`.claude/skills/cover-design/`）・エージェント指示書（`Agent.md`）・アプリ（`cover-maker.html` の `COVER_KNOWHOW`）はこの文書を基準にし、食い違う場合はこの文書を優先します。

- 姉妹文書：旧Kindle表紙メーカーのルールは `docs/cover-maker-rules.md`、漫画本文は `docs/manga-maker-rules.md`

---

## 汎用ルール（基本思想）

**表紙はアートではなく広告である。**

目的は

- スクロールを止める
- クリックされる
- 内容が伝わる
- 購入候補に入る

ことである。

デザインの美しさだけで判断しない。
常に**「この表紙は売れるか」**で判断すること。

---

## ルール1：タイトル最優先

Kindleは小さなサムネイルで見られる。
表紙で最も大きく見えるべきものは**タイトル**。
**タイトルが読めない表紙は失敗。**

視線誘導の順序：

```
タイトル
↓
主役
↓
感情フック
↓
補足情報
```

---

## ルール2：感情フックを探せ

本の内容から**読者の感情が最も動く要素**を探す。

| ジャンル | 感情フックの例 |
|---|---|
| 歴史人物 | 母への想い／夢／挑戦 |
| 自己啓発 | 変われる／逆転 |
| ビジネス | 稼げる／失敗回避 |
| 健康 | 不安解消／若返り |
| 旅行 | 憧れ／非日常 |

**表紙は情報ではなく感情を売る。**

---

## ルール3：読者メリットを探す

読者は本を買うのではない。**未来を買う。**

表紙内に

- 何が得られるか
- 何が分かるか

を配置する。

---

## ルール4：情報量を調整する

**情報不足の例**：綺麗な人物＋綺麗な背景で終わり → 内容不明。

**情報過多の例**：タイトル＋帯＋帯＋帯＋バッジ＋吹き出し＋説明文 → ごちゃごちゃで読めない。

**理想**：

```
タイトル
感情フック
読者メリット
以上
```

---

## ルール5：表紙タイプを自動判定

本を分析し、どの表紙型が最適か判定する。

| 型 | タイプ | 例 | 重視するもの |
|---|---|---|---|
| A型 | 人物ドラマ型 | 歴史漫画・自伝・小説 | 人物・感情 |
| B型 | 問題解決型 | ビジネス・健康・副業 | ベネフィット |
| C型 | 権威型 | 専門書・投資 | 信頼 |
| D型 | 憧れ型 | 旅行・ライフスタイル | 世界観 |

---

## 核心ルール：画像の評価基準（最重要・必ず入れる）

画像を評価するときは

1. タイトルは読めるか
2. 何の本か分かるか
3. 誰向けか分かるか
4. 感情が動くか
5. クリックしたくなるか

を評価する。

**美しいだけの表紙を高評価にしない。**
**サムネイルで3秒以内に内容が伝わる表紙を高評価にする。**

---

## 出力フロー（最終形）

表紙設計は必ず次の順で出力する。
案は複数並べず、プロとして判断した**最適解1案に絞る**（ユーザー決定事項・2026-07-15）。

```
STEP0：売れる要素分析
STEP1：感情フック抽出
STEP2：表紙タイプ判定
STEP3：レイアウト設計
STEP4：画像生成プロンプト生成
STEP5：Canva仕上げ指示
```

各STEPの内容：

- **STEP0 売れる要素分析** — この本だけの固有価値・読者ターゲット・読者が得る未来（メリット）・クリックしたくなる要素
- **STEP1 感情フック抽出** — 最も感情が動く要素を抽出（ルール2の表を参考に）。情報より感情を優先
- **STEP2 表紙タイプ判定** — A〜D型のどれか、理由付きで判定（ルール5）
- **STEP3 レイアウト設計** — 視線誘導（タイトル→主役→感情フック→補足情報）・配色（3色以内：ベース＋文字＋アクセント）・情報量（ルール4の理想形）・人物の顔や主役に文字をかぶせない
- **STEP4 画像生成プロンプト生成** — 文字なし版と文字入り版の両方。サイズは1600×2560px（Kindle推奨）。日本語の文字は入れない前提（Canvaで後載せ）
- **STEP5 Canva仕上げ指示** — タイトル・サブタイトル・著者名の配置、帯の有無、バッジ案、配色、フォント。画像生成AIの文字は信用しない

---

## 共通の約束事

- タイトル・サブタイトル・コピー・帯・バッジの文言に句点（。）を付けない
- 色数は基本3色以内（ベース色＋文字色＋アクセント色）
- 表紙サイズは 1600×2560px（Kindle推奨）
- 生成画像に日本語の文字は入れない。文字は後からCanvaで載せる
