🎨 AI漫画メーカー
✓ 保存済み
🗺️ スタート

絵が描けなくても大丈夫。
AIを使えば6ステップで漫画が完成します。

各ステップをクリックして作業を始めましょう。入力内容はブラウザに自動保存されます。

🛠️ ツールの役割分担
ツール役割
Gemini / GPT作る・量産する（ストーリー・コマ割り・画像プロンプト・修正）
NotebookLM設定を記憶して一貫性を保つ（キャラ・背景・構成のブレ防止）
Nano Banana / Anifusion / GPT画像を生成する（キャラシートを添付して見た目を固定）
⚠️ よくある失敗： STEP1を雑にしていきなり画像生成から始めてしまう。ストーリーとキャラが固まっていないと、NotebookLMに具体的な指示が出せません。必ずSTEP1から順番に進めましょう。
🗑️ 最初からやり直す

入力した内容（ストーリー・キャラ・背景など）をすべて一括で消去して、まっさらな状態に戻します。新しい漫画を作り始めるときに使ってください。

STEP 1

ストーリーを設計する

漫画の骨組みをつくる最重要ステップです。1問ずつ答えていけばOK。いつでも前の質問に戻れます。

🔵 Gemini 🟢 ChatGPT
🎯 誰に届ける？（Q1 / 7）
💡 読者は「みんな」ではなくたった一人に絞るほど刺さります。実在の知人を思い浮かべてOK。日常は導入シーンの舞台に、きっかけは物語が動き出す着火点になります。この人があなたの漫画の主人公の原型です。
😰 ①悩み｜導入で「これ私だ」（Q2 / 7）
📖 物語の冒頭。読者が「あるある！」と共感して引き込まれる表面の困りごとを3〜5個。
💭 ②-1 本音｜中盤で「わかる…！」（Q3 / 7）
📖 物語の心臓。建前ではなく、人には言えない泥臭い本音。ここがそのまま主人公の心の声（モノローグ）になります。
🔑 ②-2 葛藤｜壁とカギ（Q4 / 7）
📖 本音から理想への橋渡し。ここが空だと「都合よすぎ」な話に。主人公がつまずき（壁）、気づいて前へ進む（カギ）。

        

        
👆 このプロンプトをGemini / GPTに貼ると、壁・カギの候補が出ます。気に入ったものを上の欄に書きましょう。

      

    

    

      
      
    

  


  
  

    

      
🌅 ③理想の未来｜結末で「こうなれる！」（Q5 / 7）

      
📖 物語の結末。読者に「自分もこうなれる」という希望を渡します。実用・集客漫画なら、これがそのまま商品の価値・伝えたいことになります。

      

        
        
      

      

        
        
      

    

    

      
      
    

  


  
  

    

      
🏗️ 構成の「型」を選ぶ（Q6 / 7）

      
💡 ストーリーの骨格となる構成パターンを1つ選びましょう。迷ったら「起承転結」でOK。

      

        
        
        
        
      

    

    

      
      
    

  


  
  

    

      
📚 章立て｜背骨を章に分ける（Q7 / 7）

      
💡 ここまでの①悩み → ②本音・壁・カギ → ③理想を何章に分けるかを決めます。迷ったら4章（悩み／本音／壁・カギ／理想）が一番シンプルです。

      

    

    

      
      
    

  


  
  

    

      
🎉 ストーリー設計シート完成！

      
入力内容をもとに、NotebookLM の「ソース（資料）」に追加するシートを生成します。

      
📚 これはNotebookLMの【ソース（資料）】に追加して保存するものです。
💬 チャット欄に質問として打つものではありません（ただ資料棚に置くだけ）。
※コマ割りや画像プロンプトの「依頼文」（STEP4・5）は、逆にチャット欄に打ちます。

      
      

        

          
          

        

        
📚 使い方：「コピー」→ NotebookLMの「＋ソースを追加」→「テキスト」に貼る（またはGoogle Docsに保存してそのDocを追加）。質問は不要、保存するだけ。

      

      
✅ 生成できたら、次のSTEP2でキャラクターと背景を決めましょう。

    

    

      
      
    

  







  

    
STEP 2

    
キャラ・背景を固定する

    
キャラの見た目と背景を固定します。ここで決めた基本プロンプトがSTEP4でNotebookLMに渡す「素材」になります。

    

      🔵 Gemini
      🟢 ChatGPT
    

  


  

    
🖌️ ① 画風・スタイル（全キャラ・全背景 共通）

    

      
      
    

    
💡 ここで決めた画風が、すべての出力に統一して反映されます（全キャラクター・全背景・コマ割り画像・一括生成すべて）。タッチがバラつかないよう、最初にここを決めましょう。

  


  
👥 登場人物は複数登録できます。主人公のほか、相手役・脇役などを追加しましょう。各コマで「誰が出るか」を指定すると、対応する人物の設定が自動で使われます。
💡 四面図を作る理由：画像生成時にキャラシート（四面図）を参考画像として添付すると、コマごとの見た目のブレを防げます。


  


  

  
🏠 背景も複数登録できます。「自宅・会社・カフェ」など、漫画によく出てくる場所を2〜3個ぶん用意しておくと、コマごとに使い分けられます。


  


  

  

    
    
  







  

    
STEP 3

    
NotebookLMにソースを入れる

    
STEP1・2で作った素材をNotebookLMに登録します。ここが整うとSTEP4以降のAI生成の精度が大きく上がります。

    

      📓 NotebookLM
    

  


  

    
📋 NotebookLMに入れる素材（漫画制作バイブル）


    
📗 この作品ごとに用意するもの

    
    
      
    • 
      
    • 
      
    • 
    


    
📚 一度作れば毎回使い回せる｜汎用素材（あなたのバイブル）

    
    
      
    • 
      
    • 
      
    • 
    


    
💡 📗作品ごと素材は毎回作ります。📚汎用素材は一度NotebookLMに入れれば次の作品でも使い回せます。特に四面図は表情集・ポーズ集とセットで初めてキャラが固定されます。

  


  

    
⚡ STEP1・2の内容をまとめてコピー

    
STEP1〜2の入力内容をNotebookLMのソース用にまとめて生成します。

    
    

      

        
        

      

    

  


  

    
🤖 NotebookLMの本領は「設定の番人」

    
💡 NotebookLMが本当に得意なのは 設定の一貫性キープ だけ。変換・プロンプト生成・量産は Gemini / GPT の方が速いので、NotebookLMは"設定の番人"として使うのがコツです。

    

      
タスク任せるツールひとこと

      
        
設定の保管・一貫性チェックNotebookLM ◎ソースを記憶して「設定どおりか」を確認できる本領

        
コマ割り変換Gemini / GPTチャットの方が速い（NotebookLMでも可）

        
画像プロンプト生成・量産Gemini / GPT何度も差し替える反復はチャットがテンポ良い

        
セリフ・構成の修正Gemini / GPT細かい直しはチャットが速い

        
見た目の一貫性参照画像（画像AI）テキストでは保てない。キャラシートを添付する

      
    

  


  

    
    
  







  

    
STEP 4

    
コマ割りシナリオを作る

    
NotebookLMとGemini / GPTを使ってストーリーをコマ単位に落とし込みます。章ごとに生成してGoogle Docsに保存しましょう。

    

      📓 NotebookLM
      🔵 Gemini
      🟢 ChatGPT
      📄 Google Docs
    

  


  
📌 流れは1本道：① まずシナリオ（文章で物語）を作る → ② それをコマ割りに変換する。STEP1で作った本音・壁・カギは、一度シナリオにすることで活きてきます。


  
  

    
⚙️ この章の設定

    

      
      
    

    

      
      
    

  


  
  

    
📖 ① シナリオを作る（文章で物語を起こす）

    
まず章の物語を文章で書き起こします。ここで感情・本音をしっかり描くのが良い漫画のコツ。ページ数やコマ数はまだ気にしなくてOK。

    
    

      

        
        

      

      
👆 このプロンプトをGemini / GPTに貼り付けて実行→出力されたシナリオを下の欄に貼ってください。

    

  


  
  

    
📋 ② ページ構成（コマ割り）に変換する

    
章のシナリオを貼って変換すると、章まるごとを「1ページずつ・コマ数・構図／演出・アングル・セリフ・モノローグ・効果音・ナレーション」が整理された形に変換する依頼文ができます。


    
コマ割りの設定（任意・空欄でおまかせ）

    

      

        
        
      

      

        
        
      

    

    
💡 どちらも目安。空欄ならAIが内容に合わせてページ数を決めます。1ページ漫画なら「1」、20ページの章なら「20」など。コマ数はクライマックス＝1コマ、会話＝2〜3コマとシーンで増減してOK。


    
シナリオ出力をここに貼る

    

    
ページ構成に変換

    
    

      

        
        

      

      
👆 このプロンプトをNotebookLMに貼り付けて実行→出力をGoogle Docsに保存してください。

    

    
⚡ 4コマ・1ページなど超短い漫画なら、シナリオを飛ばして、このコマ割り変換にいきなり章メモを貼ってもOKです。

  


  

    
📄 Google Docsへの保存ガイド

    
📌 「○○漫画　制作ノート」というGoogle Docsを1冊用意して、章ごとの出力を追記していきましょう。

    

      
保存する内容タイミング

      
        
第1章コマ割りシナリオコマ割り変換の出力後

        
第2章・3章…同じ手順で繰り返す

      
    

    

      
      
    

  


  

    
    
  







  

    
STEP 5

    
画像プロンプトを生成する

    
コマ割りシナリオから画像プロンプトを作ります。2つのルートから選べます。

    

      📓 NotebookLM
      🔵 Gemini
      🟢 ChatGPT
      📄 Google Docs
    

  


  
🔀 ルートを選ぶ：
⚡ お手軽量産＝1ページを一発生成（速い・コマは固定的）
🎨 品質ルート＝1コマずつ生成してCanvaで組み立て（手間・自由なコマ割り）


  

    
📋 コマ割りシナリオを貼る（両ルート共通）

    

      
      
    

  


  

    
⚡ ルートA：お手軽量産（1ページ一括生成）

    
上のコマ割りから「1ページ漫画を一発生成するプロンプト」を作ります。画像AIに貼って、四面図を添付するだけ。

    
    

      

        
        

      

      
👆 このプロンプトを画像AI（Gemini / Nano Banana / ChatGPT画像）に貼り、キャラの四面図を参考画像として添付して生成→1ページ漫画が出ます。
⚠️ 日本語セリフが崩れたら、プロンプト末尾の「セリフを描く」を「吹き出しは空欄で」に変えて、文字はCanvaで乗せましょう。

    

  


  

    
🎨 ルートB：品質ルート（1コマずつ・NotebookLM用）

    
上のコマ割りから、各コマの画像プロンプト（アングル・ショット・光・英語）の依頼文を作ります。NotebookLMに渡してください。

    
    

      

        
        

      

      
👆 このプロンプトをNotebookLMに貼り付けて実行→出た各コマの英語プロンプトで1コマずつ生成→Canvaで組み立て。

    

  


  

    
📊 コマ管理表（構成・セリフ管理）

    
📌 画像プロンプトはGoogle Docsで管理します。この表はページ・コマ・キャラ・セリフの構成管理に使います。

⚡ スピード重視の場合はスキップOK。 完璧より完成を優先しましょう。
