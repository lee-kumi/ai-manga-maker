🎨 AI漫画メーカー
✓ 保存済み
🗺️ スタート

絵が描けなくても大丈夫。
AIを使えば6ステップで漫画が完成します。

各ステップをクリックして作業を始めましょう。入力内容はブラウザに自動保存されます。

🛠️ ツールの役割分担
ツール役割
Gemini / GPTゼロから作る（ストーリー・セリフ・大まかなコマ割り）
NotebookLM記憶して応用する（プロンプト生成・コマ割り調整・編集）
Nano Banana / Anifusion / GPT画像を生成する
⚠️ よくある失敗： STEP1を雑にしていきなり画像生成から始めてしまう。ストーリーとキャラが固まっていないと、NotebookLMに具体的な指示が出せません。必ずSTEP1から順番に進めましょう。
STEP 1

ストーリーを設計する

漫画の骨組みをつくる最重要ステップです。ここが固まると、あとのAIへの指示がぐっと楽になります。

🔵 Gemini 🟢 ChatGPT
🎯 テーマ
👤 ペルソナ（読者像）
😰 読者の悩み（3〜5個）
🔄 現在地 → ゴール（ビフォー → アフター）
→
💎 この漫画の約束（ベネフィット）
🏗️ 構成の「型」を選ぶ（LMソース②）
💡 ストーリーの骨格となる構成パターンを1つ選びましょう。選んだ型がNotebookLMへのソースになります。
📚 目次・各章の役割・読後変化
💡 ストーリー全体を3〜5章に分けて、各章のタイトル・役割・読後変化を書きましょう。迷ったら「起承転結」で4章構成にするのが一番シンプルです。
📖 NotebookLM ソースシート生成

入力内容をもとに、NotebookLM に投入するソースシートを生成します。

🤖 NotebookLMの4つの役割

    

      
役割内容

      
        
① 設定の保管庫キャラ・背景・構成ルールを記憶して一貫性を保つ

        
② コマ割り変換シナリオをコマ単位の演出指示に変換する

        
③ 画像プロンプト生成各コマの画像生成AIに渡すプロンプトを出力する

        
④ 編集・フィードバック構成改善・セリフ修正・フォーマット量産

      
    

  


  

    
    
  







  

    
STEP 4

    
コマ割りシナリオを作る

    
NotebookLMとGemini / GPTを使ってストーリーをコマ単位に落とし込みます。章ごとに生成してGoogle Docsに保存しましょう。

      🔵 Gemini
      🟢 ChatGPT
      📄 Google Docs
    

  


  
📌 使い方：まず「ルートA：いきなりコマ割り」を試してください。出力が薄い・違和感がある場合は「ルートB：シナリオから作る」に切り替えます。


  
  

    
⚡ ルートA：いきなりコマ割り（まず試す）

      📄 Google Docs
