# CLAUDE.md — AI漫画ツール集の申し送り

このリポジトリで作業するときは、まずこのファイルを読むこと。

## このプロジェクトは何か

Kindle出版のためのAI漫画・表紙づくりを支援する**ツール集**。
GitHub Pagesで公開されている：https://lee-kumi.github.io/ai-manga-maker/

各ツールは「作品を完成させる場所」ではなく、**画像生成AI（ナノバナナ・ChatGPT）やテキストAI（Claude・ChatGPT）に渡すプロンプトを設計する場所**である。

## ファイル一覧

| ファイル | 役割 |

## 作りの約束事（変えないこと）

- **1アプリ＝1つの自己完結HTMLファイル**。CSS/JSはすべてインライン。ビルド工程・外部ライブラリなし（Google Fontsのみ例外）
- **データはlocalStorageのみ**に保存。キーは `プレフィックス_作品名_項目` 形式（v2は `v2_`、表紙は `cm_`）
- 作品名に半角アンダースコア（_）は使わせない（キーが壊れるため）
- どのアプリにも**バックアップ保存／復元**（JSONファイル書き出し）を付ける
- デザイン：Noto Sans JP、カード型UI、ポータルのアクセント色は `#E8612A`
- 対象ユーザーは**初心者**。UI文言は日本語で、専門用語を避けて丁寧に
- 生成画像に日本語の文字は入れない。**文字は後からCanvaで載せる**のが全ツール共通の前提
- キャンバスサイズ：漫画本文は 1000×1414px（A4縦）、表紙は 1600×2560px（Kindle推奨）

## ルールの二重管理（最重要）

アプリ内のプロンプト生成ロジックと `docs/` の文書は**対**になっている。

- `v2.html` を変えたら → `docs/manga-maker-rules.md` も同時に更新
- `cover-maker.html` の `COVER_KNOWHOW` を変えたら → `docs/cover-maker-rules.md` も同時に更新
- 逆方向も同じ。**片方だけ変えて終わりにしない**

## 変更の流れ

1. 作業ブランチで開発・コミット
2. 動作確認（ヘッドレスChromiumで実際に操作して確認するのが望ましい）
3. push → **プルリクエスト作成 → 内容確認 → mainへマージ**
4. マージすると GitHub Pages が自動で再デプロイされて公開される

ユーザーはGit初心者。プルリクエストやマージの操作は代行し、報告は専門用語をかみくだいて伝えること。
